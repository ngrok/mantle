<PageHeader id="list">List</PageHeader>

The low-level scroll + row **primitive** behind [`ScrollableList`](/components/scrollable-list) and [`SelectableList`](/components/selectable-list): the bordered, rounded `bg-popover` viewport (styled after the [`MultiSelect`](/components/multi-select) popover) and its hover / selected row pills, plus the ARIA wiring. It is composition-based ‚Äî compose `List.Item` children under a `List.Root` (or the windowed `List.VirtualRoot`) and keep state on the rows.

Reach for it directly only when neither higher-level component fits. **Most lists want `ScrollableList`** (an action / navigation list) **or `SelectableList`** (a multi-select checkbox grid) ‚Äî they layer data, filtering, and selection on top of this primitive. Use `List` when you need the shared chrome with your own state and layout.

## Composition

`List` is a compound. `List.Root` (or `List.VirtualRoot`) renders the scroll viewport and its `List.Item` children; each row owns its own content and state.

```text showLineNumbers=false
List.Root                          (or List.VirtualRoot)
‚îî‚îÄ‚îÄ List.Item
    ‚îî‚îÄ‚îÄ (your row content ‚Äî a button, gridcells, a link, ‚Ä¶)
```

- **`Root`** renders the bordered, rounded `bg-popover` viewport and its `Item` children in normal flow ‚Äî the non-virtualized default. **Bound its height** (`max-h-*`, `h-*`, or `min-h-0 flex-1` in a flex parent) so long lists scroll.
- **`VirtualRoot`** is the drop-in windowed variant ‚Äî same `Item` children, only the visible slice mounted via [`@tanstack/react-virtual`](https://tanstack.com/virtual). See [Virtualization](#virtualization).
- **`Item`** is a single row ‚Äî a `<div>` pill that tints on hover / when `selected`. It takes its ARIA shape from the root's `semantics` (see [Semantics](#semantics)) and supports `asChild` (see [Polymorphism](#polymorphism)). Give each a React `key`.

## Semantics

`Root` and `VirtualRoot` take a `semantics` prop that chooses the collection's ARIA shape ‚Äî the tag stays a `<div>` (per WAI-ARIA the _role_, not the tag, carries the semantics), so both flavors share one implementation.

- **`"list"`** (the default) renders a `role="list"` of `role="listitem"` rows: a non-selecting list of actions or links with native tab order, plus `Arrow`‚Üë/`Arrow`‚Üì / `Home` / `End` moving focus between the rows' controls. This is what [`ScrollableList`](/components/scrollable-list) uses.
- **`"grid"`** renders a `role="grid"` of `role="row"` rows carrying `aria-selected`, driven as a single tab stop with `aria-activedescendant` navigation that works across a virtualized window. Pass `onActivate` to react to `Space` / `Enter` on the active row (a bare click on a row fires it too, deferring to nested interactive content), and `itemId` to point `aria-activedescendant` at a real element you own (e.g. a row's checkbox). This is what [`SelectableList`](/components/selectable-list) uses.

## Polymorphism

`List.Item` accepts `asChild` to render as your own element instead of a `<div>`, keeping the pill chrome and the selection wiring. Render it as an `<a>` for a navigation row, for example ‚Äî the whole pill becomes the link.

## Virtualization

`List.Root` renders every row. For a long collection (hundreds+), swap in `List.VirtualRoot`: it windows the same `Item` children via [`@tanstack/react-virtual`](https://tanstack.com/virtual), mounting only what's visible. The call site is otherwise identical ‚Äî same children, same props ‚Äî so opting in never changes how you author the list. **Bound its height** so the virtualizer has a viewport to measure. The example below has 60 rows; scroll to see them mount and unmount.

<Example>
	<VirtualExample />
</Example>

```tsx
import { cx } from "@ngrok/mantle/cx";
import { List } from "@ngrok/mantle/list";
import { useState } from "react";

const environments = Array.from({ length: 60 }, (_, index) => ({
	id: `env-${index}`,
	name: `Environment ${index + 1}`,
	region: ["us-east-1", "us-west-2", "eu-central-1", "ap-south-1"][index % 4],
}));

// `VirtualRoot` is a drop-in for `Root` ‚Äî same `Item` children, same props.
function Example() {
	const [activeId, setActiveId] = useState("env-2");

	return (
		<List.VirtualRoot semantics="list" aria-label="Environments" className="max-h-80">
			{environments.map((environment) => (
				<List.Item key={environment.id} selected={environment.id === activeId}>
					<button
						type="button"
						onClick={() => setActiveId(environment.id)}
						className={cx(
							"flex w-full cursor-pointer flex-col gap-0.5 rounded-md bg-transparent px-2 py-1.5 text-left text-sm",
							"focus-visible:ring-focus-accent focus-visible:ring-inset focus-visible:ring-2 focus-visible:outline-hidden",
						)}
					>
						<span className="text-strong font-medium">{environment.name}</span>
						<span className="text-body leading-4">{environment.region}</span>
					</button>
				</List.Item>
			))}
		</List.VirtualRoot>
	);
}
```

> **Windowing trade-off.** Off-screen rows aren't in the DOM, so Tab and browser find-in-page only reach mounted rows. Grid keyboard navigation still spans the full collection (arrows move the active row and scroll it into view), because focus stays on the collection via `aria-activedescendant` rather than on a row that could unmount.

## Accessibility

Name the collection with `aria-label` (or `aria-labelledby`) on `Root` / `VirtualRoot`. Under `"list"` semantics the rows are a `role="list"` / `role="listitem"` structure with native tab order ‚Äî put your own focusable control (a `<button>` or an `asChild` `<a>`) inside each row; `Arrow`‚Üë/`Arrow`‚Üì / `Home` / `End` also move real focus between those controls (skipping `disabled` rows, no wrap), and the focused row tints like hover instead of ringing the control. Under `"grid"` semantics the collection is a single tab stop: `Arrow` / `Home` / `End` move an active row tracked with `aria-activedescendant` (so focus never rides a row and navigation survives virtualization), skipping `disabled` rows, and `Space` / `Enter` ‚Äî or a bare click on a row (clicks on a nested control, label, or link are left to that control) ‚Äî fires `onActivate`. Pressing or clicking anywhere on a row also makes it the active row. Keys pressed while a nested tabbable control (a link, an overflow-menu button) has focus are handled by that control, not the grid ‚Äî `Enter` activates the control instead of toggling the row. Under virtualization, screen-reader counts stay true: windowed `list` rows carry `aria-posinset` / `aria-setsize`, while windowed `grid` rows carry `aria-rowindex` with `aria-rowcount` on the grid.

## API Reference

### List.Root

The non-virtualized collection. Accepts all `<div>` props, plus:

| Prop              | Type                         | Default  | Description                                                                                                            |
| ----------------- | ---------------------------- | -------- | ---------------------------------------------------------------------------------------------------------------------- |
| `semantics?`      | `"list" \| "grid"`           | `"list"` | ARIA shape for the collection and its rows.                                                                            |
| `onActivate?`     | `(index: number) => void`    |          | Called with the active row's index when it is activated under `grid` ‚Äî by `Space` / `Enter` or a bare row click.       |
| `itemId?`         | `(index: number) => string`  |          | The DOM id `aria-activedescendant` resolves to for a row index. Override to reference an element you own.              |
| `isItemDisabled?` | `(index: number) => boolean` |          | Whether the row at an index is disabled. Defaults to the row element's `disabled` prop; pass it for data-driven state. |

Bound the height (e.g. `className="max-h-80"`) so long lists scroll.

### List.VirtualRoot

The windowed collection ‚Äî same surface as `Root` (compose the same `Item` children), plus the virtualizer knobs:

| Prop                  | Type     | Default | Description                                                             |
| --------------------- | -------- | ------- | ----------------------------------------------------------------------- |
| `estimateItemHeight?` | `number` | `44`    | Estimated item height (px) used to seed the virtualizer before measure. |
| `overscan?`           | `number` | `8`     | Rows rendered beyond the visible window on each side.                   |

### List.Item

A single row. Accepts all `<div>` props (minus `role`, owned by the collection), plus:

| Prop        | Type      | Default | Description                                                                                                              |
| ----------- | --------- | ------- | ------------------------------------------------------------------------------------------------------------------------ |
| `selected?` | `boolean` | `false` | Tints the pill and, under `grid` semantics, sets `aria-selected`.                                                        |
| `disabled?` | `boolean` | `false` | Suppresses the hover tint; under `grid` semantics the row carries `aria-disabled` and grid keyboard navigation skips it. |
| `asChild?`  | `boolean` | `false` | Render the row as the provided child element (e.g. an `<a>`) instead of a `<div>`.                                       |
