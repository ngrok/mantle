<PageHeader id="scrollable-list">Scrollable List</PageHeader>

A scrollable list of clickable rows ‚Äî the action / navigation counterpart to [`SelectableList`](/components/selectable-list). Compose `ScrollableList.Item` rows directly inside a `ScrollableList.Viewport`; each is a `<button>` (with `onClick`) or your own element via `asChild` (e.g. an `<a>`), with an optional `selected` highlight. It shares `SelectableList`'s look ‚Äî styled like the [`MultiSelect`](/components/multi-select) popover, with rows that highlight on hover / selection as an inset, rounded pill inside a bordered `bg-popover` container ‚Äî and suits an account switcher or an SSO provider picker. It renders every row by default; for long lists, swap `Viewport` ‚Üí [`VirtualViewport`](#virtualization) to window them.

</Example>

## Composition

```text showLineNumbers=false
ScrollableList.Viewport   (or .VirtualViewport)
‚îî‚îÄ‚îÄ ScrollableList.Item
    ‚îú‚îÄ‚îÄ ScrollableList.ItemTitle
    ‚îî‚îÄ‚îÄ ScrollableList.ItemDescription
```

- **`Viewport`** renders the rounded, scrollable `bg-popover` container and its `Item` children (all of them ‚Äî non-virtualized). Reads each item's `selected` / `disabled` to style its row. **Bound its height** (`max-h-*`, `h-*`, or `min-h-0 flex-1` in a flex parent) so long lists scroll.
- **`VirtualViewport`** is the drop-in windowed variant ‚Äî same `Item` children, only the visible slice mounted. See [Virtualization](#virtualization).
- **`Item`** is a clickable row ‚Äî a `<button>` by default (`onClick`), or your own element via `asChild`. `selected` gives the tinted pill.
- **`ItemTitle` / `ItemDescription`** are optional styled spans for the emphasized title and de-emphasized sub-line.

## Virtualization

`ScrollableList.Viewport` renders every row ‚Äî simplest, and right for the short lists most action/navigation menus are. For a long list (hundreds+ of rows), swap in `ScrollableList.VirtualViewport`: it windows the same `Item` children via [`@tanstack/react-virtual`](https://tanstack.com/virtual), mounting only what's visible. The call site is otherwise identical ‚Äî same children, same props. (`@tanstack/react-virtual` ships with every list entrypoint; it is small ‚Äî a few kB gzipped ‚Äî and does no windowing work until a `VirtualViewport` renders.)

<Example>
	<ScrollableListVirtualExample />
</Example>

```tsx
import { ScrollableList } from "@ngrok/mantle/scrollable-list";
import { useState } from "react";

// `VirtualViewport` is a drop-in for `Viewport` ‚Äî same `Item` children, same props.
function Example() {
	const [activeId, setActiveId] = useState("account-2");

	return (
		<ScrollableList.VirtualViewport aria-label="Your accounts" className="max-h-80">
			{accounts.map((account) => (
				<ScrollableList.Item
					key={account.id}
					selected={account.id === activeId}
					onClick={() => setActiveId(account.id)}
				>
					<ScrollableList.ItemTitle>{account.name}</ScrollableList.ItemTitle>
					<ScrollableList.ItemDescription>
						{account.plan} ¬∑ {account.memberCount} members
					</ScrollableList.ItemDescription>
				</ScrollableList.Item>
			))}
		</ScrollableList.VirtualViewport>
	);
}
```

> **Windowing trade-off.** Off-screen rows aren't in the DOM, so Tab and browser find-in-page only reach mounted rows (arrow keys still span the full list ‚Äî they mount the target row before focusing it) ‚Äî the right trade for a large enough list. Prefer `Viewport` until a list is actually long.

## Accessibility

The list is a `role="list"` of `role="listitem"` rows (the tag is a `<div>` ‚Äî per WAI-ARIA the role, not the tag, carries the semantics); each `Item` is a focusable `<button>` (or your `asChild` element) inside its row. It is a **non-selecting** list, not a selection widget ‚Äî that's [`SelectableList`](/components/selectable-list)'s job. Provide an `aria-label` (or `aria-labelledby`) on the `Viewport` so the list has an accessible name. A `selected` row is announced via `aria-current`, not just tinted. Disabled rows are dimmed: a `<button>` row sets the native `disabled`, while an `asChild` row is conveyed with `aria-disabled`, removed from the tab order, and swallows clicks ‚Äî but prefer omitting or removing the `href` for a genuinely disabled link.

Keyboard: rows keep their native tab order, and `Arrow`‚Üë/`Arrow`‚Üì / `Home` / `End` also move focus between rows, skipping disabled ones (no wrap ‚Äî arrowing past the last row holds). The focused row lights up with the same tint as hover instead of drawing a focus ring on the item, matching the [`SelectableList`](/components/selectable-list) active row. Arrow navigation works under `VirtualViewport` too ‚Äî a jump scrolls the target row into view and mounts it before moving focus ‚Äî though off-screen rows still aren't reachable by Tab alone (see [Virtualization](#virtualization)).

## API Reference

### ScrollableList.Viewport

The non-virtualized container. Accepts all `<div>` props, plus:

| Prop       | Type        | Default | Description                                    |
| ---------- | ----------- | ------- | ---------------------------------------------- |
| `children` | `ReactNode` |         | `ScrollableList.Item` rows (all are rendered). |

Bound the height (e.g. `className="max-h-80"`) so long lists scroll.

### ScrollableList.VirtualViewport

The windowed container ‚Äî same surface as `Viewport` (compose the same `Item` children), plus the virtualizer knobs:

| Prop                  | Type        | Default | Description                                                             |
| --------------------- | ----------- | ------- | ----------------------------------------------------------------------- |
| `children`            | `ReactNode` |         | `ScrollableList.Item` rows (only the visible window is mounted).        |
| `estimateItemHeight?` | `number`    | `44`    | Estimated item height (px) used to seed the virtualizer before measure. |
| `overscan?`           | `number`    | `8`     | Rows rendered beyond the visible window on each side.                   |

### ScrollableList.Item

A clickable row. Accepts all `<button>` props (minus `type`), plus:

| Prop        | Type      | Default | Description                                                                                                                                                                               |
| ----------- | --------- | ------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `selected?` | `boolean` | `false` | Marks the row selected/current (tinted pill) and sets `aria-current`, so the state is announced, not just shown. Omit for a plain action/navigation list.                                 |
| `disabled?` | `boolean` | `false` | Disables the row (dimmed). `disabled` on the button; with `asChild`, `aria-disabled` + removal from the tab order, and clicks are swallowed ‚Äî prefer omitting `href` for a disabled link. |
| `asChild?`  | `boolean` | `false` | Render the row as the provided child element (e.g. an `<a>`) instead of a `<button>`.                                                                                                     |

### ScrollableList.ItemTitle / ScrollableList.ItemDescription

Styled `<span>`s for the title and de-emphasized sub-line inside an `Item`. Both accept all `<span>` props.
