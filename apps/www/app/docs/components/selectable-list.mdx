<PageHeader id="selectable-list">Selectable List</PageHeader>

A filterable, multi-select **grid** of checkbox rows — the inline (non-popover) counterpart to [`MultiSelect`](/components/multi-select) and [`Combobox`](/components/combobox). Map your data into `options` once; the list owns filtering and selection. Each row is an APG grid `role="row"` (`aria-selected`) whose cells hold a real [`Checkbox`](/components/checkbox) and a [`Choice`](/components/choice)-laid-out title + description — the pattern that lets a selectable row carry a real control and its own interactive content. The default row is fully click-to-toggle with a title and an optional de-emphasized sub-line — or render your own layout (see [Custom rows](#custom-rows)). It renders every row by default; for long lists, swap `Viewport` → [`VirtualViewport`](#virtualization). The example below has a disabled row (`staging`).

<Example>

</Example>

## Composition

`SelectableList` is a compound. `Root` owns selection + filter state and the derived filtered options; the parts below it read from that shared state.

```text showLineNumbers=false
SelectableList.Root
├── SelectableList.Filter
├── SelectableList.SelectAll
├── SelectableList.Viewport               (or .VirtualViewport)
│   └── SelectableList.Item               (optional render-prop; auto-rendered by default)
│       ├── SelectableList.ItemTitle
│       └── SelectableList.ItemDescription
└── SelectableList.Empty
```

- **`Filter`** drives a case-insensitive substring match over each option's `label`. Omit it for a non-filterable list.
- **`SelectAll`** reflects and toggles the selection of the **currently filtered** options (checked / indeterminate / unchecked), reusing [`selectAllChecked`](/components/checkbox#select-all-helper).
- **`Viewport`** renders the rows as a `role="grid"` of `role="row"` rows (each with `role="gridcell"` cells for the checkbox and the content). It's styled like the [`MultiSelect`](/components/multi-select) popover: a bordered, rounded `bg-popover` container whose rows highlight on hover / selection as an inset, rounded pill (the selected row picks up the accent tint, alongside its checked checkbox). Give it an `aria-label` (or `aria-labelledby`) so the grid has an accessible name, and **bound its height** (`max-h-*` / `h-*`, or `min-h-0 flex-1` inside a flex parent). It renders every row — non-virtualized.
- **`VirtualViewport`** is the drop-in windowed variant — same rows and props, only the visible slice mounted. See [Virtualization](#virtualization).
- **`Empty`** renders its children in place of the viewport when no options match the filter.

Mark an option `disabled` to render its row dimmed and exclude it from toggling and "select all" (the `legacy` row below, and one in the list at the top of this page):

## Custom rows

The default row (a title over a description) is a convenience. To render **your own layout**, pass a render-prop child to a viewport and compose a `SelectableList.Item` — add icons, badges, even interactive controls (the grid pattern permits a row to hold its own buttons/links). `Item` reads its selection and disabled state from the list by `value`, so you only describe the content. `SelectableList.ItemTitle` renders a [`Choice.Label`](/components/choice) (the real label wired to the row's checkbox) and `SelectableList.ItemDescription` a `Choice.Description`.

<Example>
	<CustomRowsExample />
</Example>

```tsx
import { SelectableList } from "@ngrok/mantle/selectable-list";
import { KeyIcon } from "@phosphor-icons/react/Key";
import { useState } from "react";

const options = [
	{ value: "onboarding", label: "Onboarding Key", description: "ng-3FbZthp***xcc3" },
	{ value: "prod", label: "prod", description: "ng-3FaThZL***8xiA" },
	{ value: "staging", label: "staging", description: "ng-7HkLmNp***1q2w" },
];

function Example() {
	const [selected, setSelected] = useState<string[]>(["prod"]);

	return (
		<SelectableList.Root options={options} value={selected} onValueChange={setSelected}>
			<SelectableList.Viewport aria-label="Access keys" className="max-h-80">
				{(option) => (
					<SelectableList.Item value={option.value}>
						<div className="flex items-center gap-2">
							<KeyIcon className="text-muted size-4 shrink-0" weight="duotone" />
							<SelectableList.ItemTitle>{option.label}</SelectableList.ItemTitle>
						</div>
						<SelectableList.ItemDescription className="ml-6">
							{option.description}
						</SelectableList.ItemDescription>
					</SelectableList.Item>
				)}
			</SelectableList.Viewport>
		</SelectableList.Root>
	);
}
```

The whole row is click-to-toggle, but it is **not** a `<label>` — it's a grid `role="row"`. A bare click anywhere toggles selection, while clicks on the checkbox, the title label, or any nested control are left to those controls (no double-toggle). So a row **can** hold interactive content (a link, an overflow menu) — each in the content cell as an ordinary sibling.

> **Why `options` is still data.** `Root` needs the full item set up front to support "select all" over the filtered set and to window rows under `VirtualViewport` (off-screen rows unmount) — neither can be derived from rendered children, since unmounted rows can't report themselves. So `options` is the lightweight **data layer** (a `value`, the filter text, and `disabled`); a viewport's render-prop is the **rendering layer**. For rich rows, look your own data up by `option.value`.

## Virtualization

`SelectableList.Viewport` renders every filtered row. For a large option set (hundreds+), swap in `SelectableList.VirtualViewport`: it windows the rows via [`@tanstack/react-virtual`](https://tanstack.com/virtual), mounting only what's visible. Same props, same render-prop — only the tag changes — and `@tanstack/react-virtual` is pulled into your bundle only when you use it. The example below has 60 options; scroll to see rows mount and unmount.

<Example>
	<VirtualViewportExample />
</Example>

```tsx
import { SelectableList } from "@ngrok/mantle/selectable-list";
import { useState } from "react";

// `VirtualViewport` is a drop-in for `Viewport` — same props, same render-prop.
function Example() {
	const [selected, setSelected] = useState<string[]>([]);

	return (
		<SelectableList.Root options={accessKeyOptions} value={selected} onValueChange={setSelected}>
			<SelectableList.Filter placeholder="Filter access keys…" aria-label="Filter access keys" />
			<SelectableList.SelectAll>Select all</SelectableList.SelectAll>
			<SelectableList.VirtualViewport aria-label="Access keys" className="max-h-80" />
			<SelectableList.Empty>No access keys found.</SelectableList.Empty>
		</SelectableList.Root>
	);
}
```

> **Windowing trade-off.** Keyboard grid navigation still works across the **full** list — arrows move the active row and scroll it into view (`aria-activedescendant` + `aria-setsize`/`aria-posinset` keep screen-reader counts true). What windowing does cost: browser find-in-page (Ctrl/⌘-F) only searches mounted rows, and there's no per-checkbox Tab stop. The right trade for a large enough list; prefer `Viewport` until a list is actually long.

## Accessibility

The viewport is an APG **grid**: a `role="grid"` of `role="row"` rows, each with `role="gridcell"` cells holding a real `<input type="checkbox">` and the title + description. This is the pattern the [WAI-ARIA grid](https://www.w3.org/WAI/ARIA/apg/patterns/grid/) prescribes for selectable rows that carry their own interactive controls (a plain listbox forbids a real checkbox or nested controls inside an option). Provide `aria-label` (or `aria-labelledby`) on the viewport so the grid has an accessible name. Each row's `aria-selected` conveys its selection alongside the checkbox's checked state; the title is a `Choice.Label`, so it names the checkbox, and the description is wired via `aria-describedby`.

Keyboard: the grid is a **single tab stop**. Tab into it and `Arrow`↑/`Arrow`↓ / `Home` / `End` move an active row (tracked with `aria-activedescendant`, so focus never rides a row and survives virtualization); `Space` / `Enter` toggles the active row. The active row shows an inset accent ring. This works identically under `VirtualViewport` — arrow keys move across the **full** option set, scrolling off-screen rows into view — because focus stays on the collection, not on a row that could unmount.

## Helpers

These pure functions back the component and are exported for custom filtering or select-all logic:

- `filterSelectableOptions(options, query)` — case-insensitive substring filter over `label`.
- `toggleSelectionValue(values, value)` — add/remove a value, returning a new array.
- `summarizeSelection(options, selected)` — `{ allSelected, someSelected }` (ignoring disabled options), ready for `selectAllChecked`.

## API Reference

### SelectableList.Root

Accepts all `<div>` props, plus:

| Prop             | Type                         | Default | Description                                            |
| ---------------- | ---------------------------- | ------- | ------------------------------------------------------ |
| `options`        | `SelectableListOption[]`     |         | The full set of selectable rows.                       |
| `value?`         | `string[]`                   |         | Controlled selected values. Pair with `onValueChange`. |
| `defaultValue?`  | `string[]`                   |         | Initial selected values for the uncontrolled case.     |
| `onValueChange?` | `(values: string[]) => void` |         | Called with the next selection whenever it changes.    |

### SelectableListOption

| Field          | Type        | Description                                                              |
| -------------- | ----------- | ------------------------------------------------------------------------ |
| `value`        | `string`    | Stable, unique selection value (flows through `value`/`onValueChange`).  |
| `label`        | `string`    | Title text; also the text matched by the filter and the accessible name. |
| `description?` | `ReactNode` | Optional de-emphasized sub-line under the title in the default row.      |
| `disabled?`    | `boolean`   | Renders the row dimmed and excludes it from toggling and "select all".   |

### SelectableList.Viewport

The non-virtualized grid. Accepts all `<div>` props (except `children`, which is the render-prop below), plus:

| Prop        | Type                                          | Default | Description                                                                                            |
| ----------- | --------------------------------------------- | ------- | ------------------------------------------------------------------------------------------------------ |
| `children?` | `(option: SelectableListOption) => ReactNode` |         | Render-prop for custom row content, called per filtered option. Defaults to a title + description row. |

### SelectableList.VirtualViewport

The windowed grid — same surface as `Viewport`, plus the virtualizer knobs:

| Prop                 | Type                                          | Default | Description                                                            |
| -------------------- | --------------------------------------------- | ------- | ---------------------------------------------------------------------- |
| `children?`          | `(option: SelectableListOption) => ReactNode` |         | Render-prop for custom row content (only the visible window mounts).   |
| `estimateRowHeight?` | `number`                                      | `44`    | Estimated row height (px) used to seed the virtualizer before measure. |
| `overscan?`          | `number`                                      | `8`     | Rows rendered beyond the visible window on each side.                  |

### SelectableList.Item

A selectable grid row (`role="row"`) for custom viewport rendering. Accepts all `<div>` props, plus:

| Prop        | Type        | Default            | Description                                                                                   |
| ----------- | ----------- | ------------------ | --------------------------------------------------------------------------------------------- |
| `value`     | `string`    |                    | The option's selection value; selection and disabled state are read from the list.            |
| `disabled?` | `boolean`   | the option's value | Force the row's disabled state. Defaults to the matching option's `disabled`.                 |
| `children`  | `ReactNode` |                    | Row content (e.g. `ItemTitle` + `ItemDescription`, or your own; interactive content allowed). |

### SelectableList.ItemTitle / SelectableList.ItemDescription

The title and de-emphasized sub-line inside a `SelectableList.Item`. `ItemTitle` renders a [`Choice.Label`](/components/choice) (the real `<label>` wired to the row's checkbox); `ItemDescription` renders a `Choice.Description` (wired via `aria-describedby`). Both forward their respective props.
