export const streamEpoch = new Date("2026-07-14T12:00:00Z").getTime();

export const formatClock = new Intl.DateTimeFormat("en-US", {
	hour: "2-digit",
	minute: "2-digit",
	second: "2-digit",
	hour12: false,
	timeZone: "UTC",
});

export function streamPoint(tick) {
	const drift = 400 * Math.sin(tick / 240);
	const swell = 600 * Math.sin(tick / 9);
	const chop = 180 * Math.sin(tick / 2.2);
	return {
		time: new Date(streamEpoch + tick * 200),
		throughput: Math.round(2400 + drift + swell + chop),
	};
}

export function LiveAreaExample() {
	const [tick, setTick] = useState(0);
	useEffect(() => {
		const interval = setInterval(() => {
			setTick((current) => current + 1);
		}, 200);
		return () => clearInterval(interval);
	}, []);
	const data = Array.from({ length: 60 }, (_, index) => streamPoint(tick + index));
	return (
		<AreaChart.Root
			data={data}
			xKey="time"
			aria-label="Live tunnel throughput"
			className="max-w-2xl"
		>
			<AreaChart.Grid />
			<AreaChart.XAxis
				tickFormat={(value) => (value instanceof Date ? formatClock.format(value) : String(value))}
			/>
			<AreaChart.YAxis />
			<AreaChart.Area dataKey="throughput" label="Throughput" curve="monotone" />
			<AreaChart.Tooltip
				labelFormat={(value) => (value instanceof Date ? formatClock.format(value) : String(value))}
			/>
		</AreaChart.Root>
	);
}

# Area Chart

A canvas-rendered area chart for showing how values — and, when stacked, a total and its parts — change over a continuous x axis. Each area fills with a 10% wash of its series color plus a 2px stroke on the band edge, rising from a shared zero baseline. Interaction ships with the Root unconditionally: a crosshair with an every-series tooltip, keyboard stepping with polite announcements, an sr-only data table, and click / <kbd>Enter</kbd> / <kbd>Space</kbd> activation. Large datasets decimate before painting.

## When to use

Reach for an `AreaChart` when the magnitude of a value over time is the story — the fill gives the value physical weight. Add `stacked` and it becomes a part-to-whole chart over time: the top edge of the stack is the total, and each band shows how much of it a series contributes.

- The fill is a deliberate 10% wash with a 2px band-edge stroke, so an area chart reads as a line chart with weight — never a saturated block that hides the grid and its siblings.
- Areas always fill from a zero baseline: the y-domain minimum is fixed at `0` at the type level (`yDomain?: [0, number | "auto"]`), so a truncated-baseline area chart is unrepresentable.
- Prefer [`LineChart`](/components/charts/line-chart) when composition doesn't matter and you are comparing the shapes of trajectories.
- Prefer [`BarChart`](/components/charts/bar-chart) when the x axis is categorical and the comparison is between discrete things, not over time.

Compose `AreaChart.Root` with the chrome parts (`Grid`, `XAxis`, `YAxis`) and one `AreaChart.Area` per series. A stacked chart of connections by protocol over two weeks:

With `stacked` on the Root, series stack in composition order from the baseline up: the first `Area` sits on the baseline and each later `Area` stacks on top of it. The tooltip lists series top-of-stack first, matching the visual order at the crosshair. Every multi-series chart must compose `AreaChart.Legend` — series identity must never rely on color-matching alone. (For a single series the legend renders nothing; the chart's title already names it.)

The tooltip, crosshair, and keyboard cursor are part of `AreaChart.Root`'s interaction contract — composing `AreaChart.Tooltip` customizes the readout, and omitting it never removes the hover layer.

## Overlapping areas

Without `stacked`, every series fills from the zero baseline and the areas overlap. The 10% wash is what keeps this legible: with two series you can read both bands and the region where they cross. `shape` gives a series' hover dot a distinct glyph (hover the crossing — EU West wears diamonds), a redundant encoding alongside color for where the bands overlap — see [Shapes](#shapes) below.

## Shapes

An area chart wears its glyphs on the legend keys and the active-point dots: a filled band has no per-point mark, so `shape` — `"circle"` (the default), `"square"`, `"triangle"`, or `"diamond"`, sized for equal visual weight — shows in the legend and on the dots that ride the band edges at the crosshair. This example pins a controlled `activeIndex` (falling back to a fixed point when the pointer leaves) so the dots stay visible without hovering: US East wears circles, EU West diamonds, AP South triangles. Move the crosshair and each dot keeps its series' glyph at every position. [Point shapes](/components/charts/scatter-plot#point-shapes) on the scatter plot page is the full family treatment.

## Live data

For live charts, append points to the end of a sliding window and let the chart do the rest. Appends animate the x domain as a continuous glide — the domain exponentially chases its target, so a new point arriving mid-glide just redirects the motion and back-to-back appends never stutter — while the plotted data itself never wobbles: new data must never make old data appear to move. As the window slides, axis tick labels (and the gridlines that share their fade) fade in and out instead of popping between tick sets, and the marks dissolve through a fade band at the plot's left edge — a scroll mask — so points exiting the window slide out smoothly instead of being clipped away. The mask activates automatically when the chart detects a sliding window and retires on its own once the stream idles. `prefers-reduced-motion` snaps everything instantly: no glide, no fades.

This demo appends one point every 200ms to a sliding 60-point window. Each point derives from a pure formula over a tick counter and a fixed epoch, so every render is deterministic; a real chart would append measurements as they arrive.

<Example>
	<LiveAreaExample />
</Example>

```tsx
import { AreaChart } from "@ngrok/mantle/area-chart";
import { useEffect, useState } from "react";

const streamEpoch = new Date("2026-07-14T12:00:00Z").getTime();

const formatClock = new Intl.DateTimeFormat("en-US", {
	hour: "2-digit",
	minute: "2-digit",
	second: "2-digit",
	hour12: false,
	timeZone: "UTC",
});

/** One deterministic point per tick: a long drift, a slow swell, and some chop. */
function streamPoint(tick: number) {
	const drift = 400 * Math.sin(tick / 240);
	const swell = 600 * Math.sin(tick / 9);
	const chop = 180 * Math.sin(tick / 2.2);
	return {
		time: new Date(streamEpoch + tick * 200),
		throughput: Math.round(2400 + drift + swell + chop),
	};
}

function LiveAreaExample() {
	const [tick, setTick] = useState(0);

	useEffect(() => {
		const interval = setInterval(() => {
			setTick((current) => current + 1);
		}, 200);
		return () => clearInterval(interval);
	}, []);

	// A sliding 60-point window: each tick appends one point and drops the oldest.
	const data = Array.from({ length: 60 }, (_, index) => streamPoint(tick + index));

	return (
		<AreaChart.Root
			data={data}
			xKey="time"
			aria-label="Live tunnel throughput"
			className="max-w-2xl"
		>
			<AreaChart.Grid />
			<AreaChart.XAxis
				tickFormat={(value) => (value instanceof Date ? formatClock.format(value) : String(value))}
			/>
			<AreaChart.YAxis />
			<AreaChart.Area dataKey="throughput" label="Throughput" curve="monotone" />
			<AreaChart.Tooltip
				labelFormat={(value) => (value instanceof Date ? formatClock.format(value) : String(value))}
			/>
		</AreaChart.Root>
	);
}
```

## Decorative charts

Set `decorative` to render the chart as a placeholder backdrop instead of real data — a synthetic area behind an empty state, an atmospheric hero graphic. A decorative chart keeps its visual rendering and animation but drops every layer that only makes sense for real data: it is hidden from assistive technology, removed from the tab order, and inert to pointer and keyboard, with no hover band, tooltip, sr-only data table, or live region. Because the values aren't information, `decorative` is mutually exclusive with the accessible-name props (`aria-label` / `aria-labelledby`) and the interaction props (`activeIndex`, `onActiveIndexChange`, `onDatumActivate`) — the type won't compile with both.

Reach for `decorative` over hand-rolled `inert` + CSS: it removes the interaction overlay at the source, so consumers never target chart internals to neutralize it. Keep the real empty-state message as a sibling of the chart (not inside it) so screen readers still announce it — the decorative chart itself is silent.

`decorative` says the values are not information; it is not a `disabled` flag (a chart is not a form control). For a chart with real data that should stay readable through its data table but skip pointer/keyboard exploration, that's a different, not-yet-shipped concern — don't reach for `decorative`.

<Example>

</Example>

## Composition

Compose the parts of an `AreaChart` together to build your own:

```text showLineNumbers=false
AreaChart.Root
├── AreaChart.Grid
├── AreaChart.XAxis
├── AreaChart.YAxis
├── AreaChart.Area (one per series)
├── AreaChart.ReferenceLine
├── AreaChart.Tooltip
├── AreaChart.Legend
└── AreaChart.CopyButton
```

- **`Root`** owns the data, the scales, the canvas renderer, and the entire interaction contract. Every other part must be composed inside it.
- **`Area`** registers one series per part; with `stacked` on the Root, areas stack in composition order with the first series at the baseline.
- **`Grid`**, **`XAxis`**, **`YAxis`**, and **`ReferenceLine`** register chart chrome that the Root paints on canvas. Omit a part to omit that chrome.
- **`Tooltip`** customizes the always-on hover/keyboard readout; omitting it never removes the hover layer.
- **`Legend`** renders series names keyed by color swatches as real DOM in flow below the plot. Always compose it on multi-series charts; it renders nothing for a single series.

## Accessibility

Every interactive chart must have an accessible name: the Root requires exactly one of `aria-label` or `aria-labelledby`, enforced at the type level. Prefer `aria-labelledby` referencing a visible title when the chart sits under a heading — no drift-prone duplicated text. The name lands on the focusable interaction overlay — the chart's single named element (the canvas itself is hidden from assistive technology so the chart is never announced twice). [Decorative charts](#decorative-charts) are the exception: they carry no real data, so they take no accessible name and are hidden from assistive technology entirely.

The plot is a single tab stop with a keyboard point cursor:

| Key                                       | Action                                                     |
| ----------------------------------------- | ---------------------------------------------------------- |
| <kbd>Tab</kbd>                            | Focus the plot.                                            |
| <kbd>←</kbd> / <kbd>→</kbd>               | Step between data points.                                  |
| <kbd>Page Up</kbd> / <kbd>Page Down</kbd> | Step by larger jumps.                                      |
| <kbd>Home</kbd> / <kbd>End</kbd>          | Jump to the first / last data point.                       |
| <kbd>Enter</kbd> / <kbd>Space</kbd>       | Activate the current data point (fires `onDatumActivate`). |
| <kbd>Escape</kbd>                         | Dismiss the point cursor and tooltip.                      |

Keyboard stepping announces the x value and every series' value at it through a debounced polite live region, so rapid arrowing never floods the screen-reader queue.

Every chart also renders an sr-only data table twin of the plotted data, bounded to the first 150 rows with a caption stating the summarization when the data is larger. Tooltips enhance, they never gate: every plotted value is reachable through the keyboard point cursor or the table.

The twin doubles as the chart's machine-readable surface: date row headers carry the exact ISO timestamp in `<time dateTime>` and value cells carry the raw number in `data-value`, so agents and scrapers read precise data from the DOM without reverse-engineering locale formatting — assistive tech still hears the human-formatted text, and the twin itself is addressable at `data-slot="area-chart-data-table"`.

## API Reference

`AreaChart.Area`, `AreaChart.Grid`, `AreaChart.XAxis`, `AreaChart.YAxis`, and `AreaChart.ReferenceLine` are renderless: they register configuration with the Root, which paints everything onto a single canvas. They accept no `className`, `style`, or other DOM props. `AreaChart.Tooltip` and `AreaChart.Legend` accept `div` props — the legend renders a real DOM element in flow, and the tooltip forwards its `div` props onto the tooltip element the Root renders. `AreaChart.CopyButton` also renders real DOM — an icon button accepting `button` props.

### AreaChart.Root

The root of an area chart: owns the data, the scales, the canvas renderer, and the entire interaction contract. Renders a `<div>` wrapping the canvas plot and is generic over `TDatum`, the row type of `data`.

All props from [div](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/div#attributes), plus:

| Prop                  | Type                                | Default       | Description                                                                                                                                                                                                                                                                 |
| --------------------- | ----------------------------------- | ------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `data`                | `readonly TDatum[]`                 |               | Rows of data; each `AreaChart.Area` reads one numeric key per row.                                                                                                                                                                                                          |
| `xKey`                | `Extract<keyof TDatum, string>`     |               | The key of each row's x value.                                                                                                                                                                                                                                              |
| `aria-label`          | `string`                            |               | The chart's accessible name (interactive charts). Exactly one of `aria-label` or `aria-labelledby` is required — enforced at the type level. Forbidden when `decorative`.                                                                                                   |
| `aria-labelledby`     | `string`                            |               | The id of a visible element that names the chart. Preferred over `aria-label` when the chart sits under a visible heading. Forbidden when `decorative`.                                                                                                                     |
| `decorative?`         | `boolean`                           | `false`       | Render as a placeholder backdrop, not real data: hidden from assistive tech, out of the tab order, inert to pointer/keyboard, no data table or live region. Mutually exclusive with the accessible-name and interaction props. See [Decorative charts](#decorative-charts). |
| `xScale`              | `"time"` \| `"linear"` \| `"point"` | auto-detected | Override the auto-detected x scale (`Date` values → `"time"`, numbers → `"linear"`, strings → `"point"`).                                                                                                                                                                   |
| `yDomain`             | `[0, number \| "auto"]`             | `[0, "auto"]` | Override the automatic y maximum. The minimum is always the zero baseline.                                                                                                                                                                                                  |
| `stacked`             | `boolean`                           | `false`       | Stack the composed series in composition order, the first series at the baseline.                                                                                                                                                                                           |
| `animate`             | `boolean`                           | `true`        | Animate enter/update transitions. `prefers-reduced-motion` always wins.                                                                                                                                                                                                     |
| `pending`             | `boolean`                           | `false`       | Hold the previous render at reduced opacity while fresh data loads — never swap a chart that has rendered once for a skeleton.                                                                                                                                              |
| `activeIndex`         | `number \| null`                    |               | Controlled active datum index (`null` = none). Omit for uncontrolled hover.                                                                                                                                                                                                 |
| `onActiveIndexChange` | `(index: number \| null) => void`   |               | Notified when hover or keyboard stepping moves the active index.                                                                                                                                                                                                            |
| `onDatumActivate`     | `(event: ChartDatumEvent) => void`  |               | Notified on datum activation: a mark click, or <kbd>Enter</kbd>/<kbd>Space</kbd> on the focused chart.                                                                                                                                                                      |

`ChartDatumEvent` is importable from `@ngrok/mantle/area-chart`: `{ index: number; xValue: string | number | Date; datum: object; dataKey: string | null }` (`datum` is the original row — narrow it to your row type), where `dataKey` names the series nearest the pointer on mouse activation and is `null` for keyboard activation.

### Chart color tokens

Series without an explicit `color` claim the ordered `--color-chart-1` … `--color-chart-8` tokens in composition order. The palette is validated for colorblind-safe adjacency and at least 3:1 contrast against the chart surface in all four themes (light, dark, high-contrast light, high-contrast dark). A claimed slot is sticky per `dataKey` for the chart's lifetime — filtering series in and out never repaints the survivors, so color follows the entity, never its rank. The ninth and later series all paint with `--color-chart-other`: fold the long tail into a single "Other" series rather than inventing more hues.

| CSS Variable          | Description                                                                                 |
| --------------------- | ------------------------------------------------------------------------------------------- |
| `--color-chart-1`     | The first categorical series slot.                                                          |
| `--color-chart-2`     | The second categorical series slot.                                                         |
| `--color-chart-3`     | The third categorical series slot.                                                          |
| `--color-chart-4`     | The fourth categorical series slot.                                                         |
| `--color-chart-5`     | The fifth categorical series slot.                                                          |
| `--color-chart-6`     | The sixth categorical series slot.                                                          |
| `--color-chart-7`     | The seventh categorical series slot.                                                        |
| `--color-chart-8`     | The eighth categorical series slot.                                                         |
| `--color-chart-other` | The fold slot for the ninth-and-later series — group the tail into a single "Other" series. |

Custom `color` strings are an escape hatch, not a shortcut: a raw hex value does not adapt across themes, and any custom palette must be re-validated for colorblind-safe adjacency and surface contrast. A series that carries good or bad meaning — an error rate, a success rate — should wear the semantic status colors, never a categorical slot.
