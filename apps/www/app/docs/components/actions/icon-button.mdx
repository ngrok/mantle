# Icon Button

Initiates an action, such as completing a task or submitting information. Renders only a single icon as children with an accessible, screen-reader-only label.

Every `IconButton` states what it means: `appearance` (visual weight) and `intent` (tone) are **required** — there are no implicit defaults. `intent="neutral"` is the workhorse tone for most icon buttons; reach for `"danger"` on destructive actions.

## Type

Like [`Button`](/components/actions/button#type), `IconButton` defaults to **`type="button"`** rather than the native `submit`, so it never accidentally submits a surrounding `<form>`. Opt in with `type="submit"` (or `type="reset"`) when the button should submit or reset a form. When `asChild` is used, `type` has no effect and is not forwarded to the child.

## isLoading

`isLoading` determines whether or not the icon button is in a loading state, default `false`. Setting `isLoading` will replace the icon with a spinner. It will also disable user interaction with the button and set `aria-disabled`.

## Polymorphism

When you want to render _something else_ as an `IconButton`, you can use the `asChild` prop to compose. This is useful when you want to splat the `IconButton` styling onto a `react-router` `Link`. Keep in mind that when you use `asChild` the `type` prop will **NOT** be passed to the child component.

<Example>
	<IconButton
		appearance="outlined"
		intent="neutral"
		asChild
		label="prestige worldwide"
		icon={<GlobeIcon />}
	>
		<Link to={href("/base/colors")} />
	</IconButton>
</Example>

```tsx
import { IconButton } from "@ngrok/mantle/button";
import { GlobeIcon } from "@phosphor-icons/react/Globe";
import { Link, href } from "react-router";

<IconButton
	appearance="outlined"
	intent="neutral"
	asChild
	label="prestige worldwide"
	icon={<GlobeIcon />}
>
	<Link to={href("/base/colors")} />
</IconButton>;
```

## API Reference

The `IconButton` accepts the following props in addition to the [standard HTML button attributes](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/button).

| Prop         | Type                                           | Default    | Description                                                                                                                                                                                                                                                                                                                |
| ------------ | ---------------------------------------------- | ---------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `appearance` | `"ghost"` \| `"outlined"`                      |            | The visual style of the `IconButton`. **Required** — there is no default, so every call site states the visual weight it means.                                                                                                                                                                                            |
| `asChild`    | `boolean`                                      | `false`    | Use the `asChild` prop to compose the `IconButton` styling and functionality onto alternative element types or your own React components.                                                                                                                                                                                  |
| `icon`       | `ReactNode`                                    |            | The icon to render inside the button. When `isLoading` is `true`, the icon is automatically replaced with a spinner.                                                                                                                                                                                                       |
| `intent`     | `"accent"` \| `"danger"` \| `"neutral"`        |            | The tone of the `IconButton` — the purpose its color communicates to the user: `"neutral"` for the workhorse tone (most icon buttons), `"accent"` for deliberate brand emphasis, and `"danger"` for a destructive or irreversible action. **Required** — there is no default, so every call site states the tone it means. |
| `isLoading`  | `boolean`                                      | `false`    | Determines whether or not the icon button is in a loading state. Setting `isLoading` will replace the icon with a spinner. It will also disable user interaction with the button and set `aria-disabled`.                                                                                                                  |
| `label`      | `string`                                       |            | The accessible label for the icon. This label will be visually hidden but announced to screen reader users, similar to alt text for img tags.                                                                                                                                                                              |
| `size`       | `"xs"` \| `"sm"` \| `"md"` \| `"lg"` \| `"xl"` | `"md"`     | The size of the `IconButton`, controlling its box size. Shared scale with `Button` — the same size name renders the same box height. Only the box scales; the icon glyph stays the same size.                                                                                                                              |
| `type`       | `"button"` \| `"reset"` \| `"submit"`          | `"button"` | The behavior of the `IconButton` when activated. Defaults to `"button"`, so an `IconButton` inside a `<form>` will **not** submit it — pass `type="submit"` to submit the form. When `asChild` is used, `type` has no effect and is not forwarded to the child.                                                            |

The `IconButton` also exposes the following CSS variables as part of its API.

| CSS Variable                  | Default    | Description                                                                                                                                                     |
| ----------------------------- | ---------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--icon-button-border-radius` | `0.375rem` | The border radius of the `IconButton`. Wrappers can set it to slot icon buttons into their chrome — `ButtonGroup`'s panel appearance tightens it to `0.125rem`. |
