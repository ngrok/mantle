<PageHeader id="sidebar">Sidebar</PageHeader>

A composable, collapsible app-navigation sidebar. `Sidebar.Root` owns the state and renders no DOM;
`Sidebar.Nav` renders the panel â€” inline on desktop, collapsing to a skinny
[icon rail](#collapsing-to-the-icon-rail), and a left-side [Sheet](/components/overlays/sheet) below the
root's `mobileBreakpoint`; and `Sidebar.Trigger` toggles it from anywhere under the root, which is what
keeps the sidebar decoupled from your app shell: place the trigger in an
[App Layout](/layouts/app-layout) header without the shell knowing the sidebar exists.

The sidebar is routing-agnostic. Compose router links with `asChild` on `Sidebar.ItemButton` and drive
`current` from your router's location. The demo below runs in its own framed document, exactly like a real
app: switch the preview to the mobile viewport to see the sheet presentation, use the trigger in the header
to collapse and expand the panel, and toggle the notice to see the shell's full-window
[`AppLayout.Notice`](/layouts/app-layout) strip push everything â€” sidebar included â€” down.
Click into the frame and press <kbd>âŒ˜B</kbd>/<kbd>Ctrl+B</kbd> â€” the shortcut binds to the frame's own
window, not this page. The demo narrows `mobileBreakpoint` to `md` so the desktop panel survives the
preview's desktop and tablet widths; ngrok's dashboards keep the `lg` default.

## Collapsing to the icon rail

On desktop the panel collapses to a skinny icon rail that keeps every row's leading icon visible and
clickable: each row becomes a square chip around its icon, group labels fade out in place â€” their rows are
retained, so icon groups stay visually separated and every icon keeps the exact position it has when
expanded â€” the switcher rows in the header and footer collapse to their leading visual (product icon,
account avatar), and the rail keeps the same `bg-base` surface as the expanded panel â€” no border, no
separate chrome. The rail width is the `--sidebar-width-icon` CSS variable (default `3.25rem`).

Everything in the icon rail stays in the tab order and the accessibility tree â€” labels are clipped or
visually hidden, not removed â€” so screen readers announce the full row names and lists keep their group
names. The scrollable `Sidebar.Body` always fades its overflowing edges; in the collapsed rail it also
hides its scrollbar and relies on the fade alone. Collapse and expand the demo above with the header
trigger or <kbd>âŒ˜B</kbd>/<kbd>Ctrl+B</kbd>.

[DropdownMenu](/components/overlays/dropdown-menu) submenu, wherever you compose that menu.

## Navigation anatomy

The navigation parts are plain styled elements: `Sidebar.Group` pairs an optional muted `Sidebar.GroupLabel`
with a `Sidebar.List` of rows, and links the two with `aria-labelledby` automatically so each list is
announced with its group's name. `Sidebar.ItemButton` renders a leading icon slot and a truncating label, and
`current` sets both `aria-current="page"` and the `data-current` visual state.

## Persisting the collapsed state

`Sidebar.Root` supports both uncontrolled (`defaultOpen`) and controlled (`open` + `onOpenChange`) desktop
state, and the mobile sheet mirrors that with `openMobile` + `onOpenMobileChange`. To persist the collapsed
state across visits, use the **fully controlled** form with
[`useLocalStorage`](/hooks#uselocalstorage):

> [!IMPORTANT]
> In SSR apps, `defaultOpen` + storage does not work: the uncontrolled state initializes exactly once at the
> hydration render, before the stored value is available. Controlled `open` is the reliable form.

This recipe is SSR-safe: [`useLocalStorage`](/hooks#uselocalstorage) is `useSyncExternalStore`-backed with a
server snapshot, so the server render and the hydration render agree (no hydration mismatch), and the
sidebar deliberately suppresses its collapse transition until after hydration, so a persisted-collapsed
sidebar **snaps** shut instead of animating on page load. What no localStorage recipe can do is affect the
server-rendered HTML itself â€” a persisted-collapsed sidebar paints expanded for one frame before the
post-hydration correction. If that frame matters, persist to a **cookie** instead: read it in your server
loader and pass it as `open`/`defaultOpen`, the same way the theme system solves FOUC server-side.

## Polymorphism

`Sidebar.Header`, `Sidebar.Body`, `Sidebar.Footer`, `Sidebar.Group`, `Sidebar.GroupLabel`, `Sidebar.List`,
`Sidebar.Item`, `Sidebar.ItemButton`, and `Sidebar.SwitcherButton` accept an `asChild` prop. When `true`, the
part renders its single child instead of its default element, forwarding all class names, `data-*`
attributes, and the ref onto that child. The most common uses are router links on `Sidebar.ItemButton` (shown
throughout this page) and composing `Sidebar.SwitcherButton` with `DropdownMenu.Trigger asChild` or
`Dialog.Trigger asChild`.

`Sidebar.Nav` and `Sidebar.Trigger` deliberately do not support `asChild`: the nav renders
a fixed two-element collapse structure (and a `Sheet` on mobile), and the trigger renders
a fully-wired control â€” build custom triggers with `useSidebar` instead. `Sidebar.AccountAvatar`,
`Sidebar.UserAvatar`, and `Sidebar.SwitchAccountsRadioGroup` are prop-driven leaves.

## Accessibility

- `Sidebar.Nav` renders a `<nav>` landmark named `Main` by default; override `aria-label` when your app has
  multiple navigation landmarks (breadcrumbs, pagination) or for localization.
- `Sidebar.Trigger` follows the disclosure pattern: `aria-expanded` reflects the state and `aria-controls`
  references the nav. Keep a trigger visible at every breakpoint where the sidebar can collapse â€” a collapsed
  sidebar with no visible trigger is unrecoverable.
- `âŒ˜B` / `Ctrl+B` toggles the sidebar (opt out with `keyboardShortcut={false}` on `Sidebar.Root`, e.g. when
  the app embeds a rich-text editor where `âŒ˜B` means bold).
- When the shell owns the document, pair the sidebar with a `SkipToMainLink` and a `Main`-composed content
  card so keyboard users can jump past the sidebar's tab gauntlet â€” see
  [App Layout â†’ Landmarks](/layouts/app-layout#landmarks).
- The collapsed icon rail stays fully interactive: row labels are clipped and switcher text is visually
  hidden, but both remain in the tab order and the accessibility tree, so accessible names do not change
  between states.
- `Sidebar.ItemButton` `current` renders `aria-current="page"`; `Sidebar.GroupLabel` is linked to its
  `Sidebar.List` via `aria-labelledby`.
- The mobile sheet is a modal dialog named "Sidebar" with focus trapping and `Escape` dismissal, courtesy of
  [Sheet](/components/overlays/sheet).
- The collapse/expand transitions respect `prefers-reduced-motion`.

## API Reference

### Sidebar.Root

The state owner. Renders no DOM.

| Prop                  | Type                      | Default | Description                                                                                       |
| --------------------- | ------------------------- | ------- | ------------------------------------------------------------------------------------------------- |
| `defaultOpen?`        | `boolean`                 | `true`  | Initial desktop expanded state for the uncontrolled case.                                         |
| `open?`               | `boolean`                 | â€”       | Controlled desktop expanded state. Pair with `onOpenChange`.                                      |
| `onOpenChange?`       | `(open: boolean) => void` | â€”       | Called with the next desktop expanded state.                                                      |
| `openMobile?`         | `boolean`                 | â€”       | Controlled mobile sheet open state. Pair with `onOpenMobileChange`.                               |
| `onOpenMobileChange?` | `(open: boolean) => void` | â€”       | Called with the next mobile sheet open state.                                                     |
| `mobileBreakpoint?`   | `"sm" \| "md" \| "lg"`    | `"lg"`  | Breakpoint below which `Sidebar.Nav` renders as a mobile `Sheet`.                                 |
| `keyboardShortcut?`   | `boolean`                 | `true`  | Toggle with `âŒ˜B` / `Ctrl+B`. Exact-modifier match (`âŒ˜â‡§B` passes through); set `false` to opt out. |

### Sidebar.Nav

The panel. All props from `div` plus the panel contract: `className`, `style`, `ref`, and rest props land on
the panel surface in both presentations (the desktop collapse container and the mobile `Sheet.Content`), and
`aria-label` (default `"Main"`) is forwarded to the inner `<nav>` landmark. Emits
`data-state="expanded" | "collapsed"` on desktop and `data-mobile` on the mobile sheet. (Desktop also
emits a bare `data-hydrated` once client hydration
completes â€” an internal gate that lets collapse transitions snap instead of animating when SSR state is
corrected on load.)

| CSS Variable             | Default   | Description                                                                |
| ------------------------ | --------- | -------------------------------------------------------------------------- |
| `--sidebar-width`        | `16rem`   | Expanded desktop width. Set via `Sidebar.Nav`'s `className` or `style`.    |
| `--sidebar-width-mobile` | `18rem`   | Mobile sheet width. Set via `Sidebar.Nav`'s `className` or `style`.        |
| `--sidebar-width-icon`   | `3.25rem` | Collapsed icon rail width. Set via `Sidebar.Nav`'s `className` or `style`. |

### Sidebar.Trigger

An [IconButton](/components/actions/icon-button) wired to toggle the sidebar. All `IconButton` props except
`asChild` and `children`, with sidebar defaults:

| Prop          | Type                                | Default                 | Description                                 |
| ------------- | ----------------------------------- | ----------------------- | ------------------------------------------- |
| `appearance?` | `"ghost" \| "outlined"`             | `"ghost"`               | The visual style of the trigger.            |
| `icon?`       | `ReactNode`                         | `<SidebarSimpleIcon />` | The icon rendered inside the trigger.       |
| `intent?`     | `"accent" \| "danger" \| "neutral"` | `"neutral"`             | The color intent of the trigger.            |
| `label?`      | `string`                            | `"Toggle Sidebar"`      | Accessible name; override for localization. |

### Sidebar.Header, Sidebar.Body, Sidebar.Footer

Structural containers inside `Sidebar.Nav`: a pinned header, the scrollable middle, and a pinned footer. All
props from `div` plus `asChild`.

### Sidebar.Group, Sidebar.GroupLabel, Sidebar.List, Sidebar.Item

The navigation grouping: `Group` (a `div`) pairs an optional `GroupLabel` (a `div` â€” pass `asChild` to render
a heading at a level you control) with a `List` (a `ul`) of `Item` rows (`li`). When a label is present the
group links it to the list via `aria-labelledby`. All props from their respective elements plus `asChild`.

### Sidebar.ItemButton

The interactive navigation row. Expanded spacing matches the regular sidebar; in the collapsed icon rail,
item rows keep their original square icon chip with a small offset so body and footer icons stay aligned
without text peeking from the rail. All props from `button` plus:

| Prop       | Type      | Default | Description                                                            |
| ---------- | --------- | ------- | ---------------------------------------------------------------------- |
| `asChild?` | `boolean` | `false` | Render the single child (e.g. a router link) instead of a `<button>`.  |
| `current?` | `boolean` | â€”       | Marks the current page: sets `aria-current="page"` and `data-current`. |

### Sidebar.SwitcherButton

The styled switcher row for the header (app switcher) and footer (account/user row). Its leading visual is
size-7 to match selected rail icons while the row keeps a compact 36px footprint. The control uses a custom
10px radius on all corners so product and account switchers share the same shape around their rounded tiles.
A styled button only â€” compose it with `DropdownMenu.Trigger asChild` or `Dialog.Trigger asChild`, which
provide the open state (`data-state="open"` styling responds automatically). All props from `button` plus
`asChild`.

Inside the collapsed icon rail, the row collapses to its **first child element** â€” the
leading visual â€” and the remaining children become visually hidden while staying in the accessibility tree,
so the button's accessible name is unchanged. Wrap loose text in an element (e.g. a `<span>`) so it
participates.

### Sidebar.Separator

An inset hairline between sidebar regions, aligned with the surrounding `Sidebar.Body` and `Sidebar.Footer`
padding (it deliberately does not run edge to edge). When the sidebar is expanded, those regions trim their
trailing inset to account for the adjacent app-content gutter. When it collapses to the icon rail, the
separator widens to the collapsed switcher chip width so it stays optically centered between the viewport
edge and content card. All props from [Separator](/components/structure/separator).

### Sidebar.AccountAvatar

A `size-7` rounded-square account avatar. The swatch color derives deterministically from `accountId` (stable
across sessions and devices) using the prototype account switcher palette.

| Prop          | Type                  | Default | Description                                              |
| ------------- | --------------------- | ------- | -------------------------------------------------------- |
| `accountId`   | `string \| undefined` | â€”       | Stable id; seeds the deterministic swatch color.         |
| `accountName` | `string \| undefined` | â€”       | Display name; its first 1â€“2 letters become the initials. |

### Sidebar.UserAvatar

A `size-7` circular user avatar with a silhouette fallback.

| Prop   | Type     | Default          | Description                                                   |
| ------ | -------- | ---------------- | ------------------------------------------------------------- |
| `src?` | `string` | â€”                | Profile picture URL; falls back to a silhouette when omitted. |
| `alt?` | `string` | `"Your account"` | Accessible label for the avatar.                              |

### Sidebar.SwitchAccountsRadioGroup

A pre-styled radio group of accounts for a "Switch accounts" `DropdownMenu` submenu. All props from Radix
`DropdownMenu.RadioGroup` (`value`, `onValueChange`, â€¦) plus:

| Prop       | Type                            | Default | Description                                        |
| ---------- | ------------------------------- | ------- | -------------------------------------------------- |
| `accounts` | `ReadonlyArray<SidebarAccount>` | â€”       | The accounts to render, `{ id, name, trailing? }`. |

### useSidebar

A hook returning the nearest `Sidebar.Root` state (`SidebarState`) â€” throws outside a root. Use it for custom
triggers, close-on-navigate, or a keyboard shortcut:

| Field              | Type                      | Description                                                  |
| ------------------ | ------------------------- | ------------------------------------------------------------ |
| `open`             | `boolean`                 | Desktop expanded state.                                      |
| `setOpen`          | `(open: boolean) => void` | Set the desktop expanded state.                              |
| `openMobile`       | `boolean`                 | Mobile sheet open state.                                     |
