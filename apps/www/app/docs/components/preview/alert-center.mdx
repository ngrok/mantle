A single, top-level entry point for one-to-many account alerts — usage limits, hard limits, billing
problems — and the upgrade CTAs that resolve them. It replaces a stack of independent window banners with
**one** severity-colored bar that surfaces the highest-priority alert inline (its message _and_ its
call-to-action), plus a compact count-and-caret trigger that expands the remaining alerts as full-width banners,
ranked by severity.

Compose it into [`AppLayout.Notice`](/layouts/app-layout#applayoutnotice), the single top-of-window
composition slot for app-wide messaging. The persistent account-alert bar can share the slot with another
window-level notice; the slot stacks them and pushes the sidebar and content down as one unit. Alerts are
passed as **data**, not authored children: the bar's count, its top alert, and the expanded list's ordering
are all _derived_ from the array, so aggregation is deterministic rather than depending on which banners
happen to mount. The demo runs in its own framed document — choose the count-and-caret control to expand or
collapse the full-width banners, or dismiss alerts to watch the bar re-rank.

		id: "payment-failed",

		title: "New region available: eu-west",

export function Shell() {
	const [dismissed, setDismissed] = useState<ReadonlySet<string>>(new Set());
	const showPreviewNotice = false;
	const alerts = accountAlerts.filter((alert) => !dismissed.has(alert.id));

			<AppLayout.Root className="fixed inset-0">

				<AppLayout.Notice>

					<AlertCenter.Root

					<AppLayout.Inset>
						<AppLayout.Content>{/* … */}</AppLayout.Content>
					</AppLayout.Inset>
				</AppLayout.Body>
			</AppLayout.Root>
		</Sidebar.Root>
	);
}

## How it works

- **The bar surfaces the top alert inline.** `AlertCenter.Bar` shows the highest-severity alert's icon,
  title, and call-to-action directly in the strip — so the most urgent CTA is one glance and zero extra
  clicks away. Hiding _every_ CTA behind a click is exactly what makes banners fail to convert.
- **The count-and-caret control expands the full list.** When two or more alerts are active, the bar renders
  a compact control that expands `AlertCenter.Content` beneath it. The additional alerts render as compact,
  full-width one- or two-line banners, push the app layout down, and can be collapsed by the same control.
- **Alerts are data.** Ordering is a pure function of the `alerts` array (a stable sort by severity:
  `danger` › `warning` › `important` › `info` › `success`), so a global billing alert and a page-level
  usage alert merge into one deterministic list instead of stacking in incidental order.
- **It collapses when empty.** With no alerts the bar renders nothing, mirroring `AppLayout.Notice`, so it
  can stay mounted and render conditionally.
- **It announces politely.** A persistent, visually-hidden `role="status"` live region (mounted by
  `AlertCenter.Root`) announces the top alert and how many more as alerts arrive or re-rank — a live region
  must exist before its text changes to be announced reliably. The bar itself claims no ARIA `banner`
  landmark.

## Basic usage

The consumer owns the alert data and dismissal (including any persistence, e.g. `sessionStorage`);
`AlertCenter` re-derives the bar and the list from the next `alerts` array.

<Example>
	<AlertCenterExample />
</Example>

```tsx
import { AlertCenter, type AlertCenterAlert } from "@ngrok/mantle/alert-center";
import { useState } from "react";

const accountAlerts = [
	{
		id: "payment-failed",
		intent: "danger",
		title: "Payment failed — update your card",
		description: "We couldn't charge the card ending in 4242.",
		action: { label: "Update payment method", href: "/billing" },
	},
	{
		id: "transfer-limit",
		intent: "warning",
		title: "You've used 92% of your monthly data transfer",
		action: { label: "Upgrade", href: "/billing/choose-a-plan" },
		dismissable: true,
	},
] satisfies AlertCenterAlert[];

export function Example() {
	const [dismissed, setDismissed] = useState<ReadonlySet<string>>(new Set());
	const alerts = accountAlerts.filter((alert) => !dismissed.has(alert.id));

	return (
		<AlertCenter.Root
			alerts={alerts}
			onDismiss={(id) => setDismissed((prev) => new Set(prev).add(id))}
		>
			<AlertCenter.Bar />
			<AlertCenter.Content />
		</AlertCenter.Root>
	);
}
```

## Custom banners

The alert records remain data so the center can rank and dismiss them, while both placements expose a
render-prop for the presentation. Use the normal [`Alert`](/components/feedback/alert#banners) banner parts
when a banner needs a different layout or content. `Bar` provides the top alert plus expand/collapse and
dismiss capabilities; `Content` provides every additional alert plus its dismiss capability.

```tsx
import { Alert } from "@ngrok/mantle/alert";
import { AlertCenter } from "@ngrok/mantle/alert-center";

<AlertCenter.Root alerts={alerts} onDismiss={dismiss}>
	<AlertCenter.Bar>
		{(alert, { dismiss, isExpanded, remaining, toggle }) => (
			<Alert.Root appearance="banner" intent={alert.intent} className="gap-2 py-2 pr-2">
				<Alert.Icon className="shrink-0" />
				<Alert.Content>
					<Alert.Title className="truncate">
						{alert.title} <a href="/usage">Review usage</a>
					</Alert.Title>
					{dismiss != null && <Alert.DismissIconButton label="Dismiss alert" onClick={dismiss} />}
					{remaining > 0 && (
						<Alert.ExpandButton count={remaining} expanded={isExpanded} onClick={toggle} />
					)}
				</Alert.Content>
			</Alert.Root>
		)}
	</AlertCenter.Bar>
	<AlertCenter.Content>
		{(alert, _index, { dismiss }) => (
			<Alert.Root appearance="banner" intent={alert.intent}>
				<Alert.Icon />
				<Alert.Content>
					<Alert.Title>{alert.title}</Alert.Title>
					{dismiss != null && <Alert.DismissIconButton onClick={dismiss} />}
				</Alert.Content>
			</Alert.Root>
		)}
	</AlertCenter.Content>
</AlertCenter.Root>;
```

count-and-caret control.

## Custom rows

`AlertCenter.Content` accepts a render-prop child. It receives each additional alert in ranked order (the top
alert remains in the bar), so the count and ordering stay data-driven while row rendering is fully composable.
Compose the `Alert` parts directly.

<Example>
	<AlertCenterCustomRowExample />
</Example>

```tsx
import { Alert } from "@ngrok/mantle/alert";
import { AlertCenter, type AlertCenterAlert } from "@ngrok/mantle/alert-center";

const alerts = [
	{
		id: "transfer-limit",
		intent: "warning",
		title: "Approaching your data transfer limit",
		action: { label: "Upgrade", href: "/billing/choose-a-plan" },
	},
	{
		id: "tunnel-limit",
		intent: "warning",
		title: "Tunnel limit reached",
		action: { label: "Upgrade", href: "/billing/choose-a-plan" },
	},
] satisfies AlertCenterAlert[];

<AlertCenter.Root alerts={alerts} defaultOpen>
	<AlertCenter.Bar />
	<AlertCenter.Content>
		{(alert) => (
			<Alert.Root appearance="banner" intent={alert.intent}>
				<Alert.Icon />
				<Alert.Content>
					<Alert.Title>{alert.title}</Alert.Title>
				</Alert.Content>
			</Alert.Root>
		)}
	</AlertCenter.Content>
</AlertCenter.Root>;
```

## Composition

Compose the parts of an `AlertCenter` together to build your own:

```text showLineNumbers=false

AlertCenter.Root
├── AlertCenter.Bar
└── AlertCenter.Content
```

## API Reference

### AlertCenter.Root

The state and data owner. It ranks the `alerts`, publishes them on context, and owns the expanded state of the
inline list. Compose its `Bar` inside `AppLayout.Notice`, alongside any other window-level notice.

| Prop           | Type                          | Default | Description                                                                            |
| -------------- | ----------------------------- | ------- | -------------------------------------------------------------------------------------- |
| `alerts`       | `readonly AlertCenterAlert[]` | —       | The active alerts. Count, top alert, and ordering are all derived from this array.     |
| `onDismiss`    | `(id: string) => void`        | —       | Called with an alert's `id` when its dismiss affordance is activated. You own removal. |
| `open`         | `boolean`                     | —       | Controlled expanded state of the additional-alert list.                                |
| `defaultOpen`  | `boolean`                     | —       | Uncontrolled initial expanded state of the additional-alert list.                      |
| `onOpenChange` | `(open: boolean) => void`     | —       | Called when the additional-alert list expands or collapses.                            |

### AlertCenter.Bar

The always-visible, full-width strip. Surfaces the highest-severity alert inline (icon, title, CTA) plus a
count-and-caret expansion control, and collapses to nothing when there are no alerts. It claims no ARIA `banner`
landmark; `AlertCenter.Root` provides the persistent `role="status"` live region. Forwards `className`,
`ref`, and other props to the underlying `Alert` element.

Pass a render-prop child to replace the default banner with a composition of `Alert`'s banner parts. It
receives `(alert, { dismiss, isExpanded, remaining, toggle })`.

| Prop       | Type                                                               | Default | Description                                             |
| ---------- | ------------------------------------------------------------------ | ------- | ------------------------------------------------------- |
| `children` | `(alert, { dismiss, isExpanded, remaining, toggle }) => ReactNode` | —       | Optional render-prop for a fully custom primary banner. |

### AlertCenter.Content

The inline expansion beneath the bar. It lists the alerts hidden by the bar, ranked highest-severity-first,
as full-width window banners. It renders only while expanded and forwards standard `div` props.

| Prop       | Type                                       | Default              | Description                                          |
| ---------- | ------------------------------------------ | -------------------- | ---------------------------------------------------- |
| `children` | `(alert, index, { dismiss }) => ReactNode` | Default `Alert` card | Optional render-prop for a custom additional banner. |

### AlertCenterAlert

The alert data type. A rich `ReactNode` title requires a plain-string `titleText` (used for the bar's
truncated summary and accessible names); a plain-string `title` needs no duplicate.

| Field         | Type                                                          | Description                                                        |
| ------------- | ------------------------------------------------------------- | ------------------------------------------------------------------ |
| `id`          | `string`                                                      | Stable identity — used for React keys, de-duplication, dismissal.  |
| `intent`      | `"danger" \| "important" \| "info" \| "success" \| "warning"` | The tone the alert's color communicates; flows through to `Alert`. |
| `title`       | `string \| ReactNode`                                         | The alert's headline (also the bar's inline message).              |
| `titleText`   | `string`                                                      | Required when `title` is a rich node; the plain-text form.         |
| `description` | `ReactNode`                                                   | Optional supporting copy shown under the title in the expansion.   |
| `action`      | `{ label; href }`                                             | Optional inline call-to-action link.                               |
| `dismissable` | `boolean`                                                     | When `true`, renders a dismiss affordance wired to `onDismiss`.    |
