	const [activeId, setActiveId] = useState("account-2");
	return (
		<div className="w-full max-w-md">

					<ScrollableList.Item

						selected={account.id === activeId}

						<ScrollableList.ItemTitle>{account.name}</ScrollableList.ItemTitle>

					</ScrollableList.Item>

			</ScrollableList.Root>
		</div>
	);
}

A virtualized, scrollable list of clickable rows — the action / navigation counterpart to [`SelectableList`](/components/preview/selectable-list). Compose `ScrollableList.Item` rows directly; each is a `<button>` (with `onClick`) or your own element via `asChild` (e.g. an `<a>`), with an optional `selected` highlight. It shares `SelectableList`'s look — styled like the [`MultiSelect`](/components/multi-select) popover, with rows that highlight on hover / selection as an inset, rounded pill inside a bordered `bg-popover` container — and handles long lists: an account switcher, an SSO provider picker.

</Example>

## Composition

```text showLineNumbers=false
ScrollableList.Root
└── ScrollableList.Item
    ├── ScrollableList.ItemTitle
    └── ScrollableList.ItemDescription
```

- **`Root`** virtualizes its `Item` children and renders the rounded, scrollable `bg-popover` container. Reads each item's `selected` / `disabled` to style its row. **Bound its height** (`max-h-*`, `h-*`, or `min-h-0 flex-1` in a flex parent) so virtualization engages; with an auto height it renders every row.
- **`Item`** is a clickable row — a `<button>` by default (`onClick`), or your own element via `asChild`. `selected` gives the tinted pill.
- **`ItemTitle` / `ItemDescription`** are optional styled spans for the emphasized title and de-emphasized sub-line.

## Navigation rows with `asChild`

Render rows as links by passing `asChild` and an `<a>` — the row chrome (hover pill, focus ring) applies to your element, so clicking navigates. Rows compose freely: each one here is a [`MediaObject`](/components/media-object) with the provider icon on the left and the name over a de-emphasized owner on the right — an SSO identity-provider picker, as a list rather than a stack of cards.

<Example>

</Example>


## Accessibility

The list is a `<ul role="list">` of `<li>` rows; each `Item` is a focusable `<button>` (or your `asChild` element). Provide an `aria-label` (or `aria-labelledby`) on `ScrollableList.Root` so the list has an accessible name. Disabled rows set the control's `disabled` (button) or `aria-disabled` (`asChild`) and are dimmed.

> **Preview / unstable API.** Keyboard support is the native tab order of the rendered rows; roving arrow-key navigation across the full virtualized list is a planned enhancement, so the API may change.

## API Reference

### ScrollableList.Root

Accepts all `<div>` props, plus:

| Prop       | Type        | Default | Description                                            |
| ---------- | ----------- | ------- | ------------------------------------------------------ |
| `children` | `ReactNode` |         | `ScrollableList.Item` rows. The list virtualizes them. |

Bound the height (e.g. `className="max-h-80"`) so virtualization engages.

### ScrollableList.Item

A clickable row. Accepts all `<button>` props (minus `type`), plus:

| Prop        | Type      | Default | Description                                                                           |
| ----------- | --------- | ------- | ------------------------------------------------------------------------------------- |
| `selected?` | `boolean` | `false` | Marks the row selected/active (tinted pill). Omit for a plain action/navigation list. |
| `disabled?` | `boolean` | `false` | Disables the row (dimmed; `disabled` on the button, `aria-disabled` when `asChild`).  |
| `asChild?`  | `boolean` | `false` | Render the row as the provided child element (e.g. an `<a>`) instead of a `<button>`. |

### ScrollableList.ItemTitle / ScrollableList.ItemDescription

Styled `<span>`s for the title and de-emphasized sub-line inside an `Item`. Both accept all `<span>` props.
